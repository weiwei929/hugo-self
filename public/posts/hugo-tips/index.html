Hugo 简介#

Hugo 是一个用 Go 语言编写的静态网站生成器，以其极快的构建速度而闻名。Hugo-Self 基于 Hugo 构建，继承了其所有优势。

内容管理技巧#

1. Front Matter 最佳实践#

Front Matter 是文章的元数据，建议包含以下字段：

 1---
 2title: "文章标题"
 3date: 2025-08-21T13:00:00+08:00
 4draft: false
 5tags: ["标签1", "标签2"]
 6categories: ["分类"]
 7description: "文章描述，用于SEO和摘要"
 8ShowToc: true
 9TocOpen: false
10weight: 1  # 用于排序
11---

2. 内容组织结构#

推荐的内容组织方式：

1content/
2├── posts/           # 博客文章
3│   ├── 2025/       # 按年份组织
4│   │   ├── 01/     # 按月份组织
5│   │   └── 02/
6│   └── tech/       # 按主题组织
7├── pages/          # 静态页面
8└── docs/           # 文档类内容

3. 图片管理#

方式一：使用 static 目录#

1static/
2└── images/
3    ├── 2025/

部署技巧#

1. GitHub Pages 部署#

创建 .github/workflows/hugo.yml：

 1name: Deploy Hugo site to Pages
 2
 3on:
 4  push:
 5    branches: ["main"]
 6  workflow_dispatch:
 7
 8permissions:
 9  contents: read
10  pages: write
11  id-token: write
12
13jobs:
14  build:
15    runs-on: ubuntu-latest
16    steps:
17      - name: Checkout
18        uses: actions/checkout@v4
19      - name: Setup Hugo
20        uses: peaceiris/actions-hugo@v2
21        with:
22          hugo-version: 'latest'
23          extended: true
24      - name: Build
25        run: hugo --minify
26      - name: Upload artifact
27        uses: actions/upload-pages-artifact@v2
28        with:
29          path: ./public
30
31  deploy:
32    environment:
33      name: github-pages
34      url: ${{ steps.deployment.outputs.page_url }}
35    runs-on: ubuntu-latest
36    needs: build
37    steps:
38      - name: Deploy to GitHub Pages
39        id: deployment
40        uses: actions/deploy-pages@v2

2. Netlify 部署#

创建 netlify.toml：

 1[build]
 2  publish = "public"
 3  command = "hugo --gc --minify"
 4
 5[context.production.environment]
 6  HUGO_VERSION = "0.146.0"
 7  HUGO_ENV = "production"
 8  HUGO_ENABLEGITINFO = "true"
 9
10[context.split1]
11  command = "hugo --gc --minify --enableGitInfo"
12
13[context.split1.environment]
14  HUGO_VERSION = "0.146.0"
15  HUGO_ENV = "production"
16
17[context.deploy-preview]
18  command = "hugo --gc --minify --buildFuture -b $DEPLOY_PRIME_URL"
19
20[context.deploy-preview.environment]
21  HUGO_VERSION = "0.146.0"
22
23[context.branch-deploy]
24  command = "hugo --gc --minify -b $DEPLOY_PRIME_URL"
25
26[context.branch-deploy.environment]
27  HUGO_VERSION = "0.146.0"

性能优化#

1. 图片优化#

 1<!-- 在模板中使用响应式图片 -->
 2{{ $image := .Resources.GetMatch "featured-image.*" }}
 3{{ if $image }}
 4  {{ $small := $image.Resize "600x" }}
 5  {{ $medium := $image.Resize "1200x" }}
 6  {{ $large := $image.Resize "1800x" }}
 7  <img 
 8    src="{{ $small.RelPermalink }}"
 9    srcset="{{ $small.RelPermalink }} 600w,
10            {{ $medium.RelPermalink }} 1200w,
11            {{ $large.RelPermalink }} 1800w"
12    sizes="(max-width: 600px) 100vw, 
13           (max-width: 1200px) 50vw, 
14           33vw"
15    alt="{{ .Title }}"
16    loading="lazy">
17{{ end }}

总结#

Hugo-Self 基于 Hugo 的强大功能，通过合理的配置和优化，可以创建出快速、美观、易维护的博客网站。

关键要点：

  1. 内容为王：专注于创作优质内容
  2. 结构清晰：合理组织文件和目录结构
  3. 性能优先：优化图片和资源加载
  4. 持续改进：根据需要逐步优化和自定义

掌握这些技巧，让你的 Hugo-Self 博客更加出色！
